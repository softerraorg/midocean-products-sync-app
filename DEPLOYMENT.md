# Midocean Products Sync — Cloudways Deployment Guide

This document explains everything you need to deploy, manage, and maintain the Midocean Shopify sync app on Cloudways. Read through once before you start — it will make the whole process straightforward.

---

## What This App Does

This is a Node.js application that automatically syncs products from the Midocean supplier API into your Shopify store. It runs on a schedule (daily by default) and handles product creation, variant updates, inventory levels, and product details (metafields).

- Then press `Enter`

## Part 11 — Changing What Gets Synced (config.js)

The file `src/config.js` controls all sync behaviour. To change settings, edit the file via FileZilla (open it in a text editor on your computer, make changes, re-upload), then restart the app.

### Production sync (all products):
| Setting | Value |
|---|---|
| `SYNC_CATEGORY_MODE` | `'all'` |
| `SYNC_CATEGORIES` | `''` |
| `SYNC_MASTER_CODES` | `[]` |
| `PRODUCT_LIMIT` | `null` |

### Sync only specific categories:
| Setting | Value |
|---|---|
| `SYNC_CATEGORY_MODE` | `'only'` |
| `SYNC_CATEGORIES` | `'Drink & lunchware'` (or comma-separated list) |

### Sync all except certain categories:
| Setting | Value |
|---|---|
| `SYNC_CATEGORY_MODE` | `'except'` |
| `SYNC_CATEGORIES` | `'Category Name'` |

### Sync only one specific product:
| Setting | Value |
|---|---|
| `SYNC_MASTER_CODES` | `['MO2995']` |

### Test run (first 5 products only):
| Setting | Value |
|---|---|
| `PRODUCT_LIMIT` | `5` |

> After changing `config.js`, always **re-upload the file via FileZilla** and then **run `pm2 restart shopify-sync`** via SSH.

## Part 12 — Changing the Sync Schedule

The sync schedule is set in `src/config.js`:
```
const SYNC_SCHEDULE = '0 0 * * *';
```

This uses standard cron format. Common examples:

| Schedule | Cron Expression |
|---|---|
| Every day at midnight | `'0 0 * * *'` |
| Every day at 2 AM | `'0 2 * * *'` |
| Every 6 hours | `'0 */6 * * *'` |
| Every 12 hours | `'0 */12 * * *'` |

After changing the schedule, re-upload `config.js` and restart: `pm2 restart shopify-sync`

## Part 14 — Troubleshooting

### App shows "errored" in pm2 status
Run `pm2 logs shopify-sync --lines 50 --nostream` to see the error. Common causes:
- `.env` file is missing or has a wrong value
- `node_modules/` not installed — run `npm install --production`
- A required file is missing — re-upload the `src/` folder

### Sync runs but no products appear in Shopify
- Check `SYNC_CATEGORY_MODE` and `SYNC_CATEGORIES` in `config.js` — you may be filtering to an empty result
- Check `PRODUCT_LIMIT` — make sure it is `null` for a full sync

### "Module not found" error in logs
- `node_modules/` is outdated — run `npm install --production` then `pm2 restart shopify-sync`

### Logs are empty
- The `logs/` folder is created automatically. If it is missing, the app creates it on first run.
- Make sure the app has started at least once: `pm2 status`
